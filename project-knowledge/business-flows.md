# Бизнес-процессы и пользовательские сценарии

## Карта пользовательских сценариев

### 1. Онбординг нового пользователя

```
[Лендинг /]
   ↓ "Get Started"
[Регистрация /register]
   ↓ Ввод email
   ↓ OTP на email (Resend)
   ↓ Ввод OTP → JWT token
[Настройки /settings]
   ↓ Заполнение профиля: рост, вес, возраст, пол, активность, цель
   ↓ Автоматический расчёт TDEE (формула Mifflin-St Jeor)
   ↓ Установка целевых КБЖУ
[Главная страница /meal-plan]
   ↓ Просмотр пустого дневного плана
   ↓ AI-ассистент предлагает начать с добавления завтрака
```

**Статус**: Регистрация ✅, Заполнение профиля ✅, TDEE расчёт ✅, Онбординг-тур ❌

---

### 2. Ведение дневного плана питания

```
[Meal Plan Dashboard /meal-plan]
   ├── Навигация по дням (вперёд/назад)
   ├── 4 слота: Завтрак / Обед / Ужин / Перекус
   ├── Добавление позиции:
   │    ├── [Поиск продукта] → выбрать → указать граммовку
   │    ├── [Поиск рецепта] → выбрать → указать порции
   │    └── [AI-ассистент] → описать → предложение → подтвердить
   ├── Карточка макросов (КБЖУ: факт vs цель)
   ├── Баланс макросов (кольцевые диаграммы)
   └── Трекинг веса / замеров
```

**Статус**: Полностью реализован ✅

### 9. AI-агент (сквозной сценарий)

```
[Любая страница]
   ↓ Нажать кнопку AI (floating button)
[GlobalAiAgentDialog]
   ↓ Ввод сообщения
   ↓ POST /v1/agent/chat (mode: "global")
   ↓ AI отвечает (read-only инструменты)

[На странице Meal Plan]
   ↓ Нажать AI-кнопку в секции плана
[MealPlanAssistantDialog]
   ↓ Описать что добавить
   ↓ POST /v1/agent/chat (mode: "meal_plan_page")
   ↓ AI предлагает → confirmation → добавляет в план

[На кнопке "+" в слоте Meal Plan]
[MealPlanItemDialog с AI режимом]
   ↓ "Хочу добавить 200г овсяной каши"
   ↓ POST /v1/agent/chat (mode: "meal_plan_slot")
   ↓ AI возвращает JSON proposal → confirmation → добавляет
```

## Ключевые зависимости между модулями

```
Auth
  └── создаёт User + Paddle customer

User Profile
  └── TDEE расчёт → targetCalories/macros

Product
  ├── используется в Recipe.ingredients
  ├── используется в MealPlanEntry
  └── используется в ShoppingListItem

Recipe
  └── используется в MealPlanEntry

MealPlanDay
  └── имеет MealPlanEntry (product | recipe | manual)

UserAiSubscription
  └── привязана к AiPlan
  └── контролирует квоты через AiUsageLog

Billing (Paddle)
  └── обновляет User.isPro, User.subscriptionStatus
  └── косвенно влияет на UserAiSubscription квоты
```

## Бизнес-правила

### Доступ к данным
- Продукты: PUBLIC (GLOBAL) видны всем, USER-продукты — только owner
- Рецепты: PRIVATE — только owner, PUBLIC — все
- Meal plans, Shopping lists, Self-care — только owner
- Один ShoppingList на пользователя

### Квоты AI
- Проверяются перед каждым AI-вызовом (`ensureCanExecute`)
- Записываются после каждого вызова (`recordUsage`)
- Сбрасываются каждый период (`currentPeriodStart/End`)

### Идемпотентность
- `IdempotencyKey` модель в БД
- Используется для предотвращения дублирования при повторных запросах

### TDEE формулы
- Mifflin-St Jeor (default)
- Harris-Benedict (Original и Revised)
- Owen
- Расчёт в `tdee/` модуле
